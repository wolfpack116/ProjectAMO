# ProjectAMO 페르소나 평가 킷 (범용)

> **목적:** 목업·화면·기능이 나올 때마다, 실사용자 관점에서 **"쓸 것인가 / 왜 안 쓸 것인가"**를 값싸게 1차 스크리닝한다. 지도·기상오버레이·모니터링·공항패널·브리핑 등 **전 표면에 재사용**.
> **정직한 경계:** 실제 사용자 테스트를 **대체하지 않는다**(뻔한 결함 거르는 1차 필터). 도메인 근거 없이 돌리면 페르소나가 항공 사실을 자신만만하게 **지어낸다** — E의 근거를 반드시 동봉. 페르소나는 **정합성 있는 실사용자만**(관제사·정비사 등 이 산출물 비사용자 제외).

---

## A. 실행 절차

1. **입력 3종 준비**
   - ① 평가 대상: 스크린샷 / 목업 HTML / 배포 URL / 소스 파일 경로
   - ② 도메인 근거: **E절 링크**(FAA/ICAO/한국·커뮤니티 리서치) — 환각 방지
   - ③ 루브릭: **B절**
2. **대상 표면 확인 → D 매트릭스로 돌릴 페르소나 선정** (전부 X, 관련된 것만)
3. **페르소나 1장 = 프롬프트 1회.** 각 카드(C) + 입력 3종을 주입하고 아래 지시:
   > "너는 이 카드의 사용자다. 주어진 근거 범위 안에서, 이 화면을 **네 맥락으로** 평가하라. **좋은 점 나열 금지**, **왜 안 쓸지 / 못 믿을지** 위주. 루브릭(B) 형식으로만 답하라. 근거 밖 항공 사실을 지어내지 마라(모르면 '확인 필요')."
4. **편향 감소(선택):** 같은 카드를 **모델 2–3종**(Claude/GPT/Gemini)으로 → 판정단. 또는 **병렬 서브에이전트**로 동시 실행.
5. **수합:** F 템플릿으로 페르소나별 + 교차 종합.

---

## B. 범용 루브릭 (모든 페르소나 공통)

각 페르소나는 아래만 답한다:

| 항목 | 내용 |
|---|---|
| **손이 가는 것** | 내 맥락에서 실제로 쓸 요소 (최대 3) |
| **건너뛸 것** | 나와 무관해 안 볼 요소 |
| **못 믿을 것** | 신뢰 안 가는 요소 + 이유(소스·신선도·재구성 등) |
| **채택 블로커 (1개)** ⭐ | 이거 안 고치면 안 쓴다 — 딱 하나 |
| **심각도** | 0 문제없음 · 1 사소 · 2 불편 · 3 심각(고전) · 4 사용불가 |

**채택 드라이버 점수(0–5)** — 표면 무관 공통 축:
- **신뢰**(권위·소스 명시·신선도) · **속도**(빠른 판단·도구왕복 감소) · **정합성**(내 작업/멘탈모델과 일치) · **완결성**(빠진 필수 요소 없음)

규칙: **적대적 프레이밍**(왜 안 쓸지), **근거 범위 준수**, **1블로커 강제**.

## C. 페르소나 카드 (6종)

> 필드 고정: **목표(JTBD) · 의사결정 기준 · 제약/환경 · 전문성·멘탈모델 · 거부 트리거 · 관심 없는 것**. 전기(경력 서사)는 넣지 않는다 — 판단을 흐린다.

### C-1. 고정익 IFR 조종사 (상업/에어택시)
| 필드 | 값 |
|---|---|
| 목표 | 계획 노선 지금 갈 수 있나 + 접근·교체 판단, 빠르게 |
| 의사결정 기준 | 개인/운항 minima, 접근 최저치, 연료·교체, icing/turb, 상층바람 |
| 제약/환경 | 시간압박, 태블릿/조종석, **공식 브리핑 병행**, 법적 책임 |
| 전문성/멘탈모델 | 실무 판독 능숙, 종관 차트 깊이는 낮음, FAA/ICAO 브리핑 순서 익숙 |
| **거부 트리거** | "공식·권위 소스 아니면 안 씀", "느리면 안 씀", "원문 아니고 재구성이면 갸웃" |
| 관심 없는 것 | 상층 이론, 넓은 종관 개황, 격자 원자료 |

### C-2. 고정익 VFR/GA 조종사
| 필드 | 값 |
|---|---|
| 목표 | 시계비행 가능한지, 안전하게 갈 수 있나 |
| 의사결정 기준 | 시정·운고·구름, 공역·TFR, 지형·장애물, 지상풍 |
| 제약/환경 | 셀프브리핑, 저고도, 단순 경로, 앱 1개로 끝내고 싶음 |
| 전문성/멘탈모델 | 중급, 접근절차·상층 잘 안 봄, 그래픽 선호 |
| **거부 트리거** | "너무 복잡·전문가용이면 부담", "VFR인데 IFR 정보만 잔뜩" |
| 관심 없는 것 | 상층바람·단열선도·접근 최저치·순항 단면도 |

### C-3. 회전익(헬기) 조종사
| 필드 | 값 |
|---|---|
| 목표 | 저고도 경로·이착륙지 안전, 국지 급변 파악 (EMS/해상/산악) |
| 의사결정 기준 | **저고도 바람·시어(LLWAS)**, 장애물, 국지 시정·안개, 결빙, 착륙지 기상 |
| 제약/환경 | 저고도·비정형 경로, 순항고도/TOD 개념 약함, 국지성 큼 |
| 전문성/멘탈모델 | 지형·국지 기상 민감, 광역 종관 관심 낮음 |
| **거부 트리거** | "고고도·순항 위주라 내 비행과 안 맞음", "국지 저고도 정보 없음" |
| 관심 없는 것 | FL300 상층바람, 순항 단면도, 장거리 종관 이동 |

### C-4. 운항관리사 (디스패처)
| 필드 | 값 |
|---|---|
| 목표 | 다수 편 릴리즈 서명·감독, 규정 준수, 연료·지연 최적 |
| 의사결정 기준 | 규정, 연료·대체, 전 노선 위험, 원자료 검증 가능성 |
| 제약/환경 | 데스크·다중 모니터, **자동계산 시스템 병행**, 다중 편 동시 |
| 전문성/멘탈모델 | 차트·모델 깊이 높음, 원자료 신뢰 중시 |
| **거부 트리거** | "원자료/출처 없으면 불신", "자동계산과 값 다르면 안 씀" |
| 관심 없는 것 | 과한 시각 장식, 단일 편 편향 |

### C-5. 항공기상 예보관 (우리 실사용층)
| 필드 | 값 |
|---|---|
| 목표 | 산출물 QA·해석·브리핑 제공, 모델 검증 |
| 의사결정 기준 | 원자료 정확성, 모델 vs 실황, 종관·상층·단열선도 깊이, 발표/유효시각 |
| 제약/환경 | 데스크, 다중 자료 교차, 전문 용어 자유 |
| 전문성/멘탈모델 | 최상, 차트·격자·IWXXM 깊이 |
| **거부 트리거** | "출처·시각·모델명 불명확", "원자료 접근 불가", "재구성이 원문인 척" |
| 관심 없는 것 | 과단순화, 판단 대신 결론만 |

### C-6. 교육훈련생 / 신입
| 필드 | 값 |
|---|---|
| 목표 | 이해하며 학습, 절차 익히기 |
| 의사결정 기준 | 용어·기호 이해 가능성, 무엇을 왜 보는지 |
| 제약/환경 | 용어 낯섦, 낮은 recall, 도움말 의존 |
| 전문성/멘탈모델 | 초급, 약어(TEMPO·TOD·SPECI) 모를 수 있음 |
| **거부 트리거** | "약어·기호 설명 없음", "빈 화면·다음 뭘 눌러야 할지 모름" |
| 관심 없는 것 | (거의 없음 — 다 보려 함, 대신 압도됨) |

## F. 산출 템플릿

### 페르소나별 (각 1블록)
```
## [C-n 페르소나명] — 대상: [기능/화면]
- 손이 가는 것: …
- 건너뛸 것: …
- 못 믿을 것: … (이유)
- 채택 블로커(1): …
- 심각도: n
- 드라이버: 신뢰 n / 속도 n / 정합성 n / 완결성 n
```

### 교차 종합
- **공통 블로커**(2인+ 지적) — 우선순위 = 심각도 × 빈도
- **페르소나별 고유** — 특정 사용자만의 이슈
- **상충 지점** — 한쪽엔 강점, 다른 쪽엔 약점(예: 차트 깊이 = 예보관 ○ / VFR ✕) → 조건부 노출로 해소?
- **권고** — 채택 최상위 지렛대 순

## 사용 예 (요약)
1. 새 목업 스크린샷 확보 → D로 페르소나 선정
2. 각 페르소나 카드 + 근거(E) + 루브릭(B) 주입 → 병렬 평가
3. F로 종합 → 우선순위 블로커 도출 → 설계 반영
4. (반복) 표면 바뀌면 대상 슬롯만 교체
