# 비행 전 기상 브리핑 (Pre-Flight Weather Briefing) — 설계 스펙

> 작성일: 2026-06-26 · 상태: 설계 합의 완료, 구현계획 대기
> 관련: `docs/briefing-architecture.md` (route-first 백엔드 아키텍처 초안), `EntryPoints.md` §7 (route briefing)

## 1. 목표 (Goal)

계획된 비행 경로(출발/목적지/교체공항 + 항로)에 대해, 이미 수집 중인 항공기상 데이터를 **조종사 의사결정 순서(FAA AIM)로 묶은 하나의 통합 브리핑**으로 제공한다. 산만한 개별 위젯이 아니라 "이 기상이 이 비행에 무엇을 할 것인가"에 답하는 **의사결정 도구**를 만든다.

이 프로젝트는 이미 METAR/TAF/SIGMET/AIRMET/AMOS/SIGWX/레이더/위성/낙뢰/KIM NWP(바람·기온·구름·착빙)/KTG 난류/연직 단면도/항로 검색을 갖추고 있다. 본 작업은 **새 데이터 수집이 아니라, 기존 데이터의 "조직화 + 시각화" 레이어**다.

## 2. 설계 철학 (Design Philosophy)

업계 표준 "**Decision support, not decision making**"을 따른다. (FAA AIM 7-1, AC 91-92, FRAT 가이드)

1. **해석은 적극적으로, 결정은 사용자.** 임계값·범주를 시스템이 해석해 부각(위험 강조)하되, go/no-go 결론은 내리지 않는다. PIC(기장)에게 최종 판단 권한이 있다.
2. **거짓 확정성 금지.** 예보는 본질적으로 확률적이다(TAF `TEMPO`/`PROB`, 착빙 "가능성"). 예보의 불확실성 표현을 평탄화하지 않는다. "비 14:00 정각" 같은 인위적 정밀도를 만들지 않는다.
3. **점진적 공개 (Progressive disclosure).** 해석된 결과(색상·요약)는 위에, 원본 해석 재료(원문 METAR·종관 차트)는 아래/펼침에. 사용자가 직접 검증·해석할 수 있게 둔다.
4. **종합 위험등급 칩(FRAT식 단일 등급)은 넣지 않는다.** 이질적 위험을 한 등급으로 줄이면 "종합 빨강만 아니면 OK" 식 거짓 안심을 유발한다(FAA FRAT 경고). 섹션별 색상까지만 해석한다.

## 3. 대상 사용자 / 모드 분기

VFR / IFR **둘 다**, 모드로 노출 산출물을 분기한다.

| | VFR 모드 | IFR 모드 |
|---|---|---|
| 강조 | 운고·시정·지상풍, 저고도 위험 | TAF 최저치·SIGMET·상층 |
| 전용 항목 | "VFR 비추천" 플래그(조건부), 일출·일몰/박명(야간 VFR) | 1-2-3 교체공항 필요 판정 |

## 4. 정보 구조 (Information Architecture)

레이아웃 = **상단 "한눈 요약 보드" + 아래 상세 섹션(세로 나열)**. 섹션 순서는 AIM(조종사 의사결정 흐름)을 따른다: 위험요인 → 큰 그림 → 지금 → 가는 길 → 도착할 때.

```
┌────────────────────────────────────────┐
│ 한눈 요약 보드                          │  ← 색칩만 보고 어디가 문제인지 즉시 파악
│  [위험🔴][출발🟢][도착🔴][교체🟡][노선🟠] │     칩 클릭 → 해당 섹션으로 점프
├────────────────────────────────────────┤
│ ① 위험 요약 (Adverse)                   │
│ ② 종관 (Synopsis) — 차트 썸네일+링크     │
│ ③ 현재 실황 (Current)                   │
│ ④ 노선·공역 (En-route) — 단면도 링크     │
│ ⑤ 목적지 예보 (Destination)             │
└────────────────────────────────────────┘
```

> 한눈 요약 보드에는 **종합 위험등급 칩을 두지 않는다.** 섹션/공항별 색칩만 둔다.

## 4.5 진입점 · 경로 입력 · 배치

### 진입점 (이미 존재)
사이드바 **"비행 전 브리핑"** 항목(`Sidebar.jsx`)이 이미 `route-check` 패널로 연결되고, 현재 **RouteBriefingPanel**("경로 확인 / Flight Plan")을 렌더한다(`MapView.jsx`의 `activePanel === 'route-check'`). 이 진입점을 그대로 쓴다.

### 경로 입력 (대부분 존재 + 2개 추가)
`useRouteBriefing.js` / `RouteBriefingPanel.jsx`가 이미 제공:
- **IFR**: 출발/도착공항, SID/STAR/IAP, 활주로·바람 기반 자동추천, FIR IN/EXIT.
- **VFR**: 경로 생성 + 편집 가능한 웨이포인트(고도 포함).
- 경로 geometry → `getCurrentRouteLineString()` → 백엔드.

**신규 입력 2개 추가** (브리핑에 필수):
- **교체공항(alternate)** — 선택(드롭다운). IFR 1-2-3 판정에 사용.
- **출발 예정 시각(ETD)** — 기본값 '현재'. **ETA = ETD + 경로거리 ÷ 순항속도**로 자동 산출(사용자 수정 가능). 순항속도는 기본값 + 입력. ETD/ETA는 TAF 도착구간 선택과 SIGMET 시간창 필터의 기준이 된다.

### 배치
브리핑 결과물은 **전용 넓은 뷰/페이지**로 렌더한다. 경로 검색 후 "브리핑 생성" → 지도 위/전체화면으로 넓게. (좁은 사이드 패널에 요약보드+섹션+표를 넣지 않는다.) 입력은 기존 route-check 흐름 재사용, 출력만 넓은 뷰.

## 4.6 지도 ↔ 브리핑 공존 모델 (인터랙션)

같은 비행을 **세 가지로 투영**한다: 수평(지도) · 수직(단면도) · 구조화(브리핑). 셋은 **경로·시간·고도·위험을 공유**하며, 같은 색 문법(VFR/MVFR/IFR, 🔴조우/🟡주변)을 쓴다.

- **모드 전환**: 풀 지도(탐색 — 모든 레이어·패닝) ↔ 브리핑 뷰(결정). 브리핑 뷰에서 "지도로" 버튼으로 복귀.
- **브리핑 뷰 레이아웃**: **sticky 영역[라우트 미니맵 + 순서 목차(①~⑤ 스텝)] + 스크롤되는 구조화 본문**.
- **라우트 미니맵**: 위치 표시기. 풀 지도가 아니라 **계획 경로 + 출발/도착/교체 + 매칭된 위험만**(3D 매칭으로 걸린 폴리곤). route preview 인프라(`routePreview.js`, `ROUTE_PREVIEW_*` 레이어, `fitBoundsRequest`)를 경로 bounds로 제한해 재사용.
- **순서 목차**: ①~⑤ 스텝 리스트. 클릭하면 해당 섹션으로 점프(목차 겸 내비).
- **스크롤 연동 (scrollytelling)**: `IntersectionObserver`로 화면 중앙에 들어온 항목을 활성으로 감지 → **미니맵 하이라이트 + 순서목차 활성 행 강조**(활성 행이 부드럽게 가운데로 auto-scroll, 글자 사라짐 없음). 각 브리핑 항목에 **공간 참조**(공항 좌표 / advisory 폴리곤 id / 경로 거리구간)를 부착해 구동.
- **역할 분담(중복 방지)**: 지도=탐색, 브리핑=결정 다이제스트, 단면도=수직 다리(④에서 "열기", v1 on-demand).

## 5. 섹션별 상세

### ① 위험 요약 (Adverse Conditions)
- **데이터**: SIGMET, AIRMET, 공항/급변풍 경보 (모두 수집 중).
- **내용**: 경로/공항에 영향을 주는 활성 위험을 강조 나열. 현상별 배지(착빙·난류·뇌우·산악차폐·LLWS 등). 위험은 **3D 매칭(§9)** 으로 🔴조우(내 계획고도에서 실제 통과)/🟡주변 인지(고도 밖)를 구분해 표기.
- **태풍·화산재**: 별도 수집 없이 **SIGMET 파서가 이미 인식하는 TC(태풍)/VA(화산재) 현상코드**를 전용 배지(🌀태풍/🌋화산재)로 강조. (`backend/src/parsers/iwxxm-advisory-parser.js`의 `PHENOMENON_LABELS`)

### ② 종관 (Synopsis) — 해석 재료
- **데이터**: 지상 종관일기도 + 예상일기도 (항공기상청 포털 **이미지 차트**).
- **표현**: 썸네일 + "차트 열기" 링크. **이 섹션만 "사용자가 직접 읽는 해석 재료"**다 (progressive disclosure의 '아래' 층).
- **근거**: 메인맵의 실황/모델 레이어(레이더·위성·KIM)는 "지금 무엇이 어디에"를 보여주지만, 전선·기압계와 그 **이동(추세·why)** 은 못 준다. 종관 차트가 그 공백을 메운다. 중복되는 WAFS 상층/난류 차트는 v1 제외.
- **제약**: 항공기상청에 **텍스트 종관 해설/예보문 API가 없음**을 확인했다. 따라서 우리가 전선/이동을 텍스트로 생성하지 않는다(거짓 확정성 금지). 차트 이미지의 fetch 가능 URL 여부는 구현 단계에서 확인.

### ③ 현재 실황 (Current)
- **데이터**: METAR/AMOS (수집 중), 비행범주(기존 `flight-category-processor`), 출발공항 **이륙예보**(API허브 이륙예보 엔드포인트).
- **표현**: 공항별 블록 = **제목 한 줄(공항명 + 범주 칩) + 2줄 표(항목행/값행)**. 출발/목적지/교체공항 각각.
  - 표 열: 바람 · 시정 · 운고 · 기온/노점 · 현상 · QNH (+ 출발은 이륙예보 풍/기온/기압).
  - **임계값을 넘는 칸만** 🟧주의/🟥초과 배경색. 정상 칸은 무채색.
- **제주 급변풍**: RKPC 관련 비행 시 LIDAR·LLWAS·양배풍 정보. **현재 데이터 없음 → 링크/샘플 플레이스홀더**로 두고 후속 실데이터 연동.

### ④ 노선·공역 (En-route) — 단면도 링크
- **데이터**: 경로상 SIGMET/AIRMET 교차, KIM NWP(바람·기온·구름·착빙), KTG 난류, 상층바람 — 모두 수집 중.
- **표현**: 경로 위험 **요약 밴드**(착빙대·난류·상층풍 등, **3D 매칭 §9**으로 계획고도 조우/주변 구분) + 작은 단면도 썸네일 + **"단면도 열기" 링크**.
- **단면도 통합 방식 = A (요약 + 링크).** 전체 인터랙티브 단면도는 **이미 구현된 연직 프로파일 창**(`POST /api/vertical-profile`, `VerticalProfileWindow.jsx`)을 재사용한다. 브리핑 안에 전체 차트를 인라인하지 않는다(가볍고 PDF도 깔끔).

### ⑤ 목적지 예보 (Destination)
- **데이터**: TAF (수집 중).
- **표현**: 평문 디코드 + 변화군(FM/TEMPO/BECMG/PROB)을 시간순으로. **도착 ETA 구간 강조.**
- **IFR 모드**: TAF 최저치 기반 **1-2-3 규칙으로 교체공항 필요 여부 자동 평가**해 표시(권고, 결론 아님).

### VFR 모드 전용 추가
- **"VFR 비추천" 플래그**: 임의 공항 운고/시정이 VFR 최저치 미만이면 ① 상단에 조건부 배지.
- **일출·일몰/박명 시각**: 야간 VFR 판단용 (항공기상청 해넘이출몰시각/달빛정보).

## 6. 시각화 시스템

- **표 + 임계값 색상.** 값은 텍스트(`140° 25kt G35`, `3000m`, `BKN 800ft`). 게이지·다이얼은 쓰지 않는다(과함).
- **공통 색상 문법** (전 요소 일관):
  - 🟢 정상 / VFR · 🟡 주의 / MVFR / 임계 접근 · 🔴 초과 / IFR / 위험 · 🟣 LIFR / 심각
- **원문 접근**: 각 공항/항목은 클릭 시 원문 METAR/TAF를 펼친다(progressive disclosure).
- 기존 자산 재사용: `flight-category-processor`(범주 색), `shared/weather/visual-mapper`(기상코드→한글), `WeatherIcon`, 아이콘 레지스트리.

## 7. 결과물 (Output)

- **v1: 인터랙티브 브리핑 뷰.**
- **v1.1(후속): PDF/인쇄 내보내기.** 뷰와 PDF가 **동일한 브리핑 payload 하나**를 공유하도록 v1부터 payload를 PDF-ready로 설계한다. 차트(종관·단면도)는 PDF에서 정적 이미지로.

## 8. 아키텍처 (합성 위치 = C 하이브리드)

위험 강조·AIM 정렬·임계값 판정 **로직은 백엔드**가 소유하고, 뷰/PDF는 같은 payload를 렌더만 한다. 단, **미래 전체 비전이 아니라 v1 섹션만큼만** 좁게 구현(YAGNI). `docs/briefing-architecture.md`의 "frontend should not interpret advisory rules" 원칙과 일치.

### 백엔드
- 신규 `POST /api/route-briefing` — 입력: 경로 geometry + 출발/목적지/교체공항 + flightRule + ETD/ETA + 계획고도. 출력: AIM 순서로 정렬된 섹션 + 섹션/공항/항목별 위험 플래그 + 비행범주 + (IFR)1-2-3 판정 + 차트/단면도 링크 URL.
- 기존 재사용: `briefing/route-axis.js`, `briefing/profile-composer.js`, `vertical-profile.js`(단면도), `processors/*`(metar/taf/sigmet/airmet/warning/flight-category), `iwxxm-advisory-parser.js`(TC/VA).
- 신규 모듈(제안): `backend/src/briefing/briefing-composer.js`(섹션 조립·위험 플래그), `backend/src/briefing/hazard-matcher.js`(3D 매칭: 폴리곤∩경로·고도밴드·시간, 조우/주변 판정). 임계값·매칭 규칙은 순수 함수로 분리해 테스트 가능하게.
- 이륙예보: `api-client.js`에 엔드포인트 추가 + parser/processor.

### 프론트엔드
- 기존 `features/route-briefing/`의 경로 입력 흐름(route-check 패널)을 재사용하고, **브리핑 뷰를 전용 넓은 뷰/페이지로 신규 추가**. `api/briefingApi.js`(이미 존재) 확장.
- 경로 폼에 **교체공항(선택) + ETD 입력**을 추가하고 ETA(거리÷순항속도) 자동 산출.
- 레이아웃은 `layoutTokens.css` 토큰을 따르고 `docs/ui-responsive-guidelines.md`를 준수(반응형 증빙 포함).

## 9. 데이터-경로 매칭 (3D)

매칭은 **수평(평면) 전용이 아니라 3D**다 — 위험은 3D 볼륨(advisory = 폴리곤 × FL밴드, KIM/KTG = 다층 격자)이고, 항공기는 계획 고도 프로파일(상승→순항→강하)을 따라 그 공간을 통과한다. 연직 단면도를 만든 목적이 이 수직 매칭이다.

**매칭 차원 = 수평 ∩ 수직 ∩ 시간:**

| 섹션 | 수평 | 수직 | 시간 |
|---|---|---|---|
| 공항(출발/도착/교체) | ICAO 직매칭 | — | 도착/교체 TAF는 ETA 포함 예보구간 |
| ① 위험요약 (advisory) | 경로축 ∩ 폴리곤(point-in-polygon) + 공항반경 | 계획고도선 ∩ FL밴드(`lower_fl`/`upper_fl`) | `valid_from~to` ∩ [ETD, ETA] |
| ④ 노선·공역 | KIM/KTG를 경로축 컬럼에 투영 | 계획고도선 높이에서 격자 보간 → 실제 비행고도의 착빙/난류/바람 | 구간별 ETA 최근접 예보시각(`selectNearestForecastHour`) |

**판정은 2단계 결과:**
- 🔴 **조우** — 수평·수직·시간 전부 충족(내 계획고도에서 실제 통과).
- 🟡 **주변 인지** — 수평·시간은 맞고 고도만 빗나감(컬럼엔 있으나 내 고도 밖, 상·하층).

**재사용 (이미 구현):**
- `briefing/cross-section-sampler.js` — `buildCrossSection`(KIM: 레벨별 `altFt` + 거리별 기온·spread·착빙등급·u/v)와 `buildKtgCrossSection`(난류, 고도×거리). 즉 (거리 × 고도) 필드가 이미 생성됨.
- `briefing/profile-composer.js` — `buildFlightPlanProfile`이 계획 고도선(climb/cruise/descent + TOD)을 `(거리, 고도Ft)`로 생성.
- advisory 파서가 폴리곤·FL밴드·유효시간 보유.

**신규로 만들 것 (모듈: `briefing/hazard-matcher.js` 제안):**
- 경로 샘플 ∩ advisory 폴리곤(ray casting), 계획고도선 ∩ FL밴드, 시간 overlap → 조우/주변 분류.
- cross-section 필드를 계획고도선 높이에서 보간해 비행고도의 KIM/KTG 위험 추출.
- 이 샘플러들을 **서버에서 동일 호출**(프런트 시각화용 = 브리핑 판정용), 새 데이터소스 0.

**한계 (정직하게):**
- KIM은 기압면 격자 → 고도 해상도가 수집 레벨 수에 제한, 레벨 사이 보간(얇은 층 누락 가능).
- advisory `lower_fl`/`upper_fl`가 null일 수 있음 → 수직 판정 불가 시 "주변 인지" 또는 전 컬럼 폴백.
- KTG도 이산 고도 → 계획고도에서 보간.
